# DS-хэдеры: перенос на прод (инструкция для программиста)

Документ описывает, как перенести дизайн-системные (DS) хэдеры с `pr-diz-develop.ru`
на боевой `pr-diz.ru`, **не сломав контент**. Всё, что «уезжало» в процессе,
собрано в разделе «⚠️ Чеклист граблей» — прочитай его обязательно.

---

## 1. Принцип: трогаем ТОЛЬКО хэдер

Ключевая идея — **неинвазивность**. Контент страниц (таблицы, формы, кнопки,
паддинги, фон) остаётся как был. Меняется только шапка.

- Грузим **не весь** `_bootstrap-flexim-overrides.css`, а извлечённый из него
  `flexim-ds/header.css` — там **только** правила для компонентов хэдера
  (`.flexim-*`, `.app-topbar`). Bootstrap-классы контента (`.btn`, `.table`,
  `.form-control` …) НЕ трогаются.
- Хэдер позиционируется **поверх** контента: рейка `position:fixed` слева,
  верхняя панель `position:sticky` сверху. Контент НЕ оборачивается в новый
  каркас и течёт как раньше.
- Легаси-отступ `body { padding-left: 60px }` (задел под старую фиксированную
  рейку `#left_bar`) **оставляем** — новая рейка тоже 60px и встаёт на его место.

---

## 2. Файлы

### Скопировать целиком (папка `flexim-ds/`)
- `flexim-ds/tokens.css` — CSS-переменные (цвета, размеры, тени).
- `flexim-ds/header.css` — ТОЛЬКО стили компонентов хэдера (извлечено из
  `_bootstrap-flexim-overrides.css`).
- `flexim-ds/prod-fixes.css` — позиционирование хэдера (рейка fixed, топбар sticky,
  тень) + нейтрализация конфликтов с легаси `css/main.css`.
- `flexim-ds/icons.js` — рендер иконок `data-flexim-icon` в inline-SVG.
- `flexim-ds/_dev-selfcheck.js` — самопроверка (бейдж справа-внизу). Только для
  дева; на проде можно не подключать.

### Общий каркас хэдера
- `include/_ds_header.php` — рейка (роли/активный раздел) + верхняя панель (табы
  раздела + поиск + чип пользователя). Раздел задаёт `$ds_tabs` и `$ds_search_html`,
  затем `include '_ds_header.php'`.

### Хэдеры по разделам (состав табов 1:1 со старыми)
- `include/header_zakaz_ds.php`  (Заказы)
- `include/header_pack_ds.php`   (Упаковка)
- `include/header_sklad_ds.php`  (Склад)
- `include/header_cut_ds.php`    (Резка)
- `include/header_plan_ds.php`   (План)
- `include/header_admin_ds.php`  (Админка)

Каждый переносит **ту же** серверную логику, что и старый `header_*.php` (роли,
активные табы, поиск), меняется только разметка на DS. Старые `header_*.php` можно
оставить (не мешают) — переключение идёт заменой include на странице.

---

## 3. Подключение (2 файла)

### `include/head.php` — после `main.css` добавить:
```php
<link href="<?=APPLICATION ?>/flexim-ds/tokens.css?v=<?=time() ?>" rel="stylesheet">
<link href="<?=APPLICATION ?>/flexim-ds/header.css?v=<?=time() ?>" rel="stylesheet">
<link href="<?=APPLICATION ?>/flexim-ds/prod-fixes.css?v=<?=time() ?>" rel="stylesheet">
```
Порядок важен: `main.css` → `tokens.css` → `header.css` → `prod-fixes.css`.
`?v=<?=time() ?>` — анти-кэш (см. грабли).

### `include/footer.php` — после подключения bootstrap.min.js добавить:
```php
<script src="<?=APPLICATION ?>/flexim-ds/icons.js?v=<?=time() ?>"></script>
<script>if(window.fleximIcons) window.fleximIcons.renderAll();</script>
<?php if(!empty($ds_header)): // самопроверка только на DS-страницах (dev) ?>
<script src="<?=APPLICATION ?>/flexim-ds/_dev-selfcheck.js?v=<?=time() ?>"></script>
<?php endif; ?>
```
`_ds_header.php` выставляет флажок `$ds_header = true`. Закрывать в footer ничего
не нужно — хэдер контент не оборачивает.

---

## 4. Перевод страницы на DS-хэдер

На странице раздела меняется **одна строка** — подключение хэдера:
```php
// было:
include '../include/header_zakaz.php';
// стало:
include '../include/header_zakaz_ds.php';
```
Больше на списочных страницах ничего менять не нужно: контент течёт как раньше.

---

## 5. ⚠️ Чеклист граблей (из-за них всё «уезжало»)

1. **Грузить только `header.css`, а НЕ весь `overrides.css`.** Полный overrides
   перекрашивает ВЕСЬ контент (кнопки, таблицы, фон). `header.css` содержит только
   `.flexim-*`/`.app-topbar` — контент не трогает.
2. **Не удалять `body { padding-left: 60px }` из `main.css`.** Рейка `fixed`
   встаёт на это место. Уберёшь — контент наедет на рейку.
3. **НЕ оборачивать контент в `app-shell`/`app-content`.** Обёртка добавляет свои
   паддинги/скролл и сдвигает вёрстку. Хэдер только «висит» поверх.
4. **Жёсткие абсолютные координаты под старую высоту хэдера — переделать на
   динамику.** Пример: в `plan/index.php` блок расписания был
   `position:absolute; top:100px` (подогнано под старый хэдер). При новом хэдере
   другой высоты — уезжает. Решение: считать `top` в JS от реальной высоты шапки
   (`getBoundingClientRect().bottom`), а не хардкодить. См. `FixPlanWrapperTop`
   в `plan/index.php` — там подробный комментарий.
5. **Margin collapsing.** Отступ через `margin-top/bottom` может «схлопнуться»
   через край контейнера без padding/border и не появиться. Для гарантированного
   отступа используй `padding` (не схлопывается) или задавай в JS.
6. **`css/main.css` перебивает DS по специфичности.** Правила `a:not(.btn)`
   красят ссылки/ховеры сильнее одиночного DS-класса — табы розовеют/краснеют.
   `prod-fixes.css` возвращает поведение DS более специфичными селекторами
   (`.app-topbar a.flexim-header-menu__item…`). Держи фиксы этим отдельным слоем,
   не правь `overrides.css`/`header.css` под легаси.
7. **Кэш.** Фавикон и CSS без версии браузер (особенно Safari) кэширует упорно —
   правки «не видно». Подключай статику с `?v=<?=time() ?>` (или версией), а при
   проверке жми `Cmd/Ctrl+Shift+R`.
8. **CSS-комментарий закрывается `*/`, а не `-->`.** Если случайно закрыть
   HTML-синтаксисом, весь `<style>` ниже комментария «съедается» и не применяется.

---

## 6. Как проверить, что не уехало

На дев-странице справа-внизу — бейдж самопроверки `_dev-selfcheck.js`:
`DS-хэдер: N/N ✓ как в дизайне`. Кликни — развернётся список (рейка 60px и
зафиксирована, верхний слой, тень, панель sticky, состав хэдера совпал с разделом,
активный таб, «контент не обёрнут / не заморожен»). Если 10/10 зелёное — вёрстка
на месте и контент не тронут. На проде self-check можно не подключать.

Точечно замерять удобно в DevTools-консоли: `getBoundingClientRect()` у
`.app-topbar` / `.flexim-nav-rail` / нужного блока.

---

## 7. Особый случай: раздел «План»

У Плана самая кастомная сетка (боковая «Очередь» + свои табы машин + расписание,
всё в `position:absolute` блоке). Именно он «уезжал» сильнее всего. Что сделано
(всё с комментариями прямо в `plan/index.php`):
- `FixPlanWrapperTop()` — динамический `top` расписания от низа саб-меню машин
  (грабли п.4).
- Отступы верха заданы токенами; воздух hr→контент — через JS (грабли п.5).
- Саб-меню машин оформлено как DS-меню: линия-подчёркивание на всю ширину под
  блоком (`border-bottom` у `.nav2`), активный таб — розовый сегмент на ней внизу.

При переносе Плана скопируй его inline-`<style>` и `FixPlanWrapperTop` целиком —
там всё прокомментировано.
